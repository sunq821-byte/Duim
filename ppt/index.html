Part 01 · 绪论 研究意义与现状
BACKGROUND · 为何研究AI Native事务管理

研究背景与意义

01 · 行业痛点
传统事务管理效率低下

用户需要适应软件的多层表单和复杂菜单，每次记账需6步操作。日程管理、消费记录、提醒设置分散在不同应用中，缺乏统一入口。

02 · 技术成熟
大语言模型能力突破

DeepSeek、Qwen等国产大模型在自然语言理解、结构化输出方面达到实用水平。Temperature=0时JSON输出稳定可靠，为AI Native应用奠定基础。

03 · 研究空白
缺乏AI Native事务方案

现有AI应用多为聊天机器人，缺乏将自然语言直接转化为结构化事务操作（记账+日程+提醒）的AI Native系统。本项目填补这一空白。

国内外研究现状：自然语言处理 + 任务自动化领域已有探索，但面向个人事务管理的AI Native应用研究较少 SIGNIFICANCE · 05

Part 03 · 关键技术与实现 Prompt Workflow 机制
PROMPT ENGINEERING · 模块化Prompt设计

单LLM + 多模块Prompt职责分离

设计原则
  • ▸Prompt模块独立文件管理，可版本控制
  • ▸每个模块单一职责：拆分 / 识别 / 提取 / 解析
  • ▸Temperature=0 确保JSON输出稳定可预测
  • ▸Few-shot示例 + JSON Schema严格约束
// Prompt 模块示例：expense_extractor SYSTEM: 你是精确的记账助手。 只输出JSON，不要任何解释。 FEW-SHOT: 输入："午饭35元" 输出: {"type":"expense","amount":35,"category":"餐饮"} SCHEMA: { "type": "expense|schedule|reminder", "amount": number | null, "category": string | null, "title": string | null, "time": string | null }
核心竞争力不是模型本身，而是 Prompt 设计 + Workflow 调度 + 输出校验 PROMPT WORKFLOW · 11
Part 03 · 关键技术与实现 混合时间解析
TIME PARSING · LLM语义 + 规则引擎

混合式自然语言时间解析

纯规则方式

正则匹配固定模式，无法理解"下周三""大后天"等模糊表达。

✗ 灵活性差
✗ 拓展成本高
纯LLM方式

LLM直接输出日期时间，存在幻觉风险。复杂时间计算不稳定。

△ 灵活性好
△ 准确性不足
混合方式 ★

LLM负责语义理解与意图识别，规则引擎负责精确的时间计算与格式化。

✓ 灵活 + 精确
✓ 可解释可调试
输入
"明天下午3点"
→
LLM语义理解
{"relative_day": "+1", "hour": 15, "minute": 0}
→
规则引擎计算
2026-05-26 15:00

Part 05 · 总结与展望 项目创新点总结
INNOVATIONS REVIEW · 三项核心创新回顾

AI Native 不是口号，是工程实践

01
自然语言交互
颠覆传统表单操作模式。用户无需学习软件界面，一句话即可完成记账、日程、提醒等多类型事务操作。核心价值：降低使用门槛至零。
02
模块化 Prompt Workflow
单LLM + 多Prompt模块职责分离。每个模块独立管理、独立优化、独立测试。避免多Agent编排的复杂度，保持工程可控性。

混合式时间解析
LLM语义理解自然语言中的模糊时间表达，规则引擎精确计算。兼顾灵活性与准确性，是系统中准确率最高的模块之一。
