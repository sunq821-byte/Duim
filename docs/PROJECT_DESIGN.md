# AI Native 自然语言事务管理系统 - 项目设计文档

## 一、项目简介

### 项目名称
AI Native 自然语言事务管理系统

### 项目定位
本项目是一款基于 AI 大模型与 Agent Workflow 的智能事务管理系统。

用户无需再通过传统 APP 的“点击按钮 → 选择分类 → 填写表单 → 保存”方式进行操作，而是直接通过自然语言（文字/语音）输入需求，由 AI 自动完成：

- 意图识别
- 信息提取
- 分类打标
- 时间解析
- 数据创建
- 提醒生成

系统核心目标：

> 让软件理解用户，而不是让用户适应软件。

---

## 二、项目背景

传统记账、日程、TODO 类软件存在以下问题：

### 1. 操作路径复杂
用户需要多次点击、切换页面、填写表单。

例如记账流程：

创建 → 选择类型 → 填金额 → 选时间 → 保存

### 2. 软件逻辑固定
用户必须适应软件定义好的交互逻辑。

### 3. 无法理解自然语言
用户输入：

“昨天和朋友吃火锅花了230”

传统软件无法自动解析：
- 金额
- 分类
- 时间
- 备注

### 4. AI时代下交互方式落后
当前大模型已经具备：
- 自然语言理解
- 结构化提取
- 时间推理
- 多任务拆分

传统事务类 APP 仍采用表单式交互，效率低下。

---

## 三、项目目标

本项目目标是构建一个：

# AI Native 事务操作系统

用户只需一句话：

“今天午饭35，明天下午3点开会提醒我”

系统即可自动完成：

- 识别多个事务
- 自动分类
- 自动补全时间
- 创建账单
- 创建日程
- 返回自然语言反馈

---

## 四、核心功能设计

## 1. AI记账

用户输入：

- 今天午饭35
- 昨天打车20
- 奶茶16

系统自动：

- 提取金额
- 判断分类
- 识别时间
- 判断收入/支出
- 创建账单

---

## 2. AI日程创建

用户输入：

- 明天下午3点开会提醒我
- 下周五交作业
- 每周一晨会

系统自动：

- 提取时间
- 创建提醒
- 生成重复周期

---

## 3. 多事务拆分

用户输入：

“今天午饭35，晚上打车20，明天开会提醒我”

系统自动拆分为：

- 餐饮支出
- 交通支出
- 会议日程

---

## 4. 自然语言修改

用户输入：

- 把昨天打车20改成25
- 删除今天午饭记录
- 取消明天会议

系统自动完成修改与删除。

---

## 5. AI统计分析（后期）

系统自动：

- 月度消费统计
- 分类占比分析
- 异常消费提醒
- 时间规划建议

## 六、Agent Workflow 设计

本项目采用“伪多Agent + 模块化职责”设计。

并非多个模型互相聊天，而是：

# 单模型 + 多职责工作流

---

## 核心模块

### 1. Input Agent
负责接收用户输入。

支持：
- 文本输入
- 语音转文字（后期）

---

### 2. Intent Agent
负责识别用户意图：

- expense
- income
- schedule
- modify
- delete
- query

---

### 3. Extract Agent
负责提取结构化信息：

- 金额
- 时间
- 分类
- 备注
- 地点

---

### 4. Classification Agent
负责自动分类：

- 餐饮
- 交通
- 娱乐
- 学习
- 工资

---

### 5. Time Parsing Agent
负责时间理解：

- 明天
- 后天下午
- 下周三
- 每月5号

---

### 6. Storage Agent
负责数据库写入。

---

### 7. Feedback Agent
负责自然语言反馈。

例如：

“已为你记录餐饮支出35元”
